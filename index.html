2. 求职进度记录

一定不要盲目投递，对于每个公司的每个岗位，都应该在个人简历基础上做针对性的优化；且一阶段内不要投递过多，更不要投递过少，应该有所跟进，在投递中慢慢迭代简历。

3.求职日志（可选）

有些小伙伴会把自己的求职经历记录下来，在论坛开贴与大家交流。

这时可以浏览一下别人的帖子，发现和竞争对手的差距。

可以去知乎、牛客等地发帖或看帖。

3.3初试

确定求职意向

一定要有求职意向，不打没准备的战。求职意向事关你对某一岗位的理解，决定了简历的基本写法，也决定你之后还需要做哪些准备；也可以有多个相近的求职意向。

求职意向要根据个人特点，更重要的是市场环境确定，一切选择都应该在足够的信息基础下做出。

可以询问院系老师学长学姐们的去向，更多的是通过求职渠道及搜索引擎甚至书籍了解职业信息；个人性格可以询问身边朋友的看法，避免过于主观。

可以利用下面的清单做一个自我检查。

了解求职流程

不同岗位的求职流程是有差异的，比如工科岗位及技术岗等是没有群面的，是否准备按需决定。

好多岗位都有笔试的，一定要提前了解是否有笔试，笔试题型是什么，提前练习。

一般都有专业面和HR面，即使是HR面都需要重视。

建立求职档案

了解求职渠道

1. 亲友推荐

如果家里或者亲友家有一定资源，可以通过他们获得一定的信息和机会。

也可以向学长学姐、老师求助。


2. 求职社群

抱团取暖群。强烈建议和相同水平的同学组队互助，同一方向更佳，一个人的能力毕竟是有限的。对于本科生而言更是如此，因为身边缺乏就业氛围，单打独斗容易松懈，也容易耗费过多精力。

求职信息群。可以加入一些推送岗位的群，比公众号还要方便些，推荐牛客。


3. 高校就业网

一定不要忘记你们学校的就业网，这里的企业有一定的意向性，有时候还能捡到来自校友的真内推。

也可以去相同水平的高校就业网看看，毕竟高校都有一定的地域限制。这里推荐简单校招，做了信息聚合。


4. 企业官网直投

大中公司都有自己的招聘网站，可以搜比如「阿里巴巴校招」。

在公众号、论坛拿到的内推码，其实与官网直投差别不大，最多是优先处理简历。官网直投和内推码都代表你有一定求职意向。


5. 招聘平台

实习生招聘网站：实习僧，刺猬实习。专注实习生招聘，反馈快。

老牌招聘网站：智联招聘，前程无忧，应届生求职网，界面不是很友好，但是信息量较大。

互联网垂直招聘：拉勾网。

直聊招聘网站：Boss 直聘。有时候对方是直属领导，跳过HR的简历筛选真的很重要。


6. 公众号

利用好公众号，公众号一般有账号认证，可以保证一定的真实性，所以该去清理下公众号啦：

企业招聘服务号。如「阿里巴巴招聘」，可以第一时间获取一手资料。

信息发布公众号。如「今日实习」「实习派」「校招派」等。

简历内容过少

深挖经历，使用PAR法则说明

使用花花绿绿的模板

可以试试超级简历

非主流的照片

及时是互联网公司也要正装照，事实上大多数岗位可以不放

过于主观，太多形容词

尽量不做自我评价，用数据说话

过于口语化（的了）

使用专业术语，特别是岗位相关术语，做好投递记录，心态要好

一份简历走天下

精细化投递，对不同公司不同岗位进行微调

必要性

1. 证明自己优秀，证明自己适合。

对岗位要有一定的了解，了解其日常工作内容与团队合作方式，体现契合；

对自己也要有详尽的了解，即能够清晰表明自己的发展路径。

2. 简历简历，写一页就好，但也不能字数过少，毕竟简历是敲门砖。

3. 给面试官一个提问的机会，即至少体现一个值得了解的点。

4. 以相关程度排序，最相关内容放在1/4处。

可选模块

1. 个人优势/自我总结

一般不写，太主观化了。转行且没有相关经历的话需要写，证明以其它事例证明自己胜任。

一定不要出现「认真负责、为人热情、沟通能力强」等描述，这部分特质需要通过经历表现。

2. 兴趣爱好

展现自己的精神状态，最好是有一定的成绩，比如坚持多久，拿过什么奖项。

3. 求职信

一般是邮箱投递用，下边会谈到。

4. 作品集

某些岗位，如新媒体运营、产品策划等强烈建议提供作品集。

可以的话和简历打包上传，也可以使用WPS云文档等分享链接，用草料生成二维码附在简历上。            

其它提示

投递与跟进

1. 简历制作工具

超级简历近来比较流行，可视化编辑，描述文字提示，自动生成一页，直接导出PDF，比较方便。

其实更建议自己用 Word 做一个，体现自己的审美和技能。也可以用 Markdown。

2. 拓展阅读

几个案例。

正确看待求职焦虑，简历排版不要犯这些错误。

3. 向专业人士求助

通过公众号、知乎、微博等认识职场人士。

领英、朋友等职场社交软件认识校友。

「在行」等付费咨询服务。

1. 投递方式

如果有越过HR的渠道最好，要保证信息真实性。就业网、Boss直聘，论坛直达面试官的链接。

官网、平台、公众号网申，复制本地简历的内容。如果使用 Chrome 的话推荐牛客简历助手。

邮箱投递，注意邮件礼节。注意事项点此。

2. 精细化投递

最好是有一个意向清单，对每个公司都有大致的了解。

投递每个岗位时，对简历进行微调，求职信也要在草稿基础上针对岗位优化。

也不要把鸡蛋只放在十个篮子里，多次投递，多次机会。

做好记录，避免失忆。

3. 保持一个节奏

不要一股脑儿投递好多简历，分批投递，可以是一周10个、一天1个。

及时跟进投递情况，比如7天后登入网站了解是否通过筛选。

根据反馈情况持续优化简历。

必选模块

1. 文件标题/文件名

一定要规范，可以是【岗位】姓名-毕业院校-年级


2. 个人信息

姓名。如果有英文文字可以写，但面试时注意答应。

联系电话。中间分隔，如123-4567-8910.

邮箱。常用的邮箱，小众邮箱包括校园邮箱可能被收垃圾箱；账号名不要数字，签名要正式。

照片。一般不需要照片，除非销售岗等对形象有要求的岗位；如果长相不是加分项就不放了。

籍贯、血型、邮编，政治面貌一般不写，看具体岗位特点。


3. 教育背景

学校。

学历。尽量写明是本科还是硕士。

专业。绩点排名好写上，如果对口的话可以写相关课程及分数。

起始年份。一定要写，且不要用「至今」。

不一定要把教育经历放最上面。


4. 实习经历

写明起始时间、公司、部门、岗位。公司写产品名称，岗位名也可以增强相关性，但是不能造假。

写明背景、行动、成果，体现个人职责和贡献，分点呈现，用数字说话，专业术语表达。

动宾结构，不要用主语，不要“负责”到底。


5. 项目经历

社团、学生会、办公室，短期兼职，比赛竞赛，甚至课程作业都可以算项目经历。

同样需要深入挖掘，体现相关性；主要负责最好，不要几个重复经历。

可以使用四象限法则深入挖掘——

我有我知道：按时间顺序回忆；尽可能多罗列；参与过也可以

我有我不知道：求助亲友，有时候你只是忘了，或者误以为某些经历没意义

我没有我知道：去学

我没有我也不知道：求助专业人士

虽然现在一般都是网申，但准备一份简历还是很有必要的，一方面可以作为附件发送，另一方面也可以复制粘贴。

网申的简历一般是先机器通过关键词筛选，之后是招聘的工作人员花很短的时间过一下，简历的可读性及相关性也是很必要的。

笔试

了解题型

关于笔试的题型，可以在各大论坛社区比如知乎搜索到，是否有笔试也因岗位而异；

不同公司同一岗位也可能是不一样的题型。


一般有如下题型：

1. 性格测试。无对错，但要注意如实填写，会通过前后对比判定你是否诚实。

2. 行测题。有图形、数字推理，资料分析，言语理解，统计图理解等，注意图形题，很难的。

3. 专业客观题。需要对这一岗位的职责和要求有深入了解。

4. 专业主观题。需要多了解多学习，掌握一些解题框架，训练思维。


笔试准备

1. 刷题。在牛客、一起求职等网站有计算机互联网金融营销职能等行业免费的题库可以使用。

2. 多了解岗位，阅读相关资讯，拓展眼界。

一般而言，通过简历筛选和笔试，来到面试说明你已经基本胜任了，在面试时要好好表现。


知己知彼

1. 了解该公司业务，可以阅读最近的新闻；在整体环境下再看该岗位，猜测面试官可能会问什么。

2. 回看简历，对每条经历都心中有数，猜测面试官会对什么感兴趣。

3. 整理问题清单，输出答案，反复记忆，做到熟练。

4. 有条件可以对面试官进行一些了解，比如在领英上查看他的履历。


准备自我介绍

1. 自我介绍不是念简历，也不是草草说一下姓名和毕业院校就可以，也不能长篇大论没有重点。

2. 撰写一个通用的自我介绍，3分钟内，群面的话还需要1分钟的；经常更新，使其更能突出亮点。

3. 单面自我介绍一般3部分内容：我是谁，我为何能够胜任，为何我选择这家公司的这个岗位。


群面

1. 了解群面流程（同时确认自己的岗位有没有群面）

    自我介绍。1分钟内，不能太简单，也不能太秀，让大家记住你就好。

    3分钟了解试题，独立思考，列出发言提纲。

    轮流阐述，也可以谁有想法谁就发言。

    自由发言。也可以在最后十分钟模拟汇报，做补充。

    选出一位总结发言，其它成员可以补充。

    面试官提问。有时会问对自己表现的评价，对别人表现的评价，注意团队利益最重要。

2. 群面角色

    Leader 领导小组，制定讨论框架，控制讨论节奏。不要刻意抢角色，自然生成。

    Timer 时间管理，但也不能只计时，积极讨论。不要只提醒时间，告知下一步干啥。

    Reporter 方案汇报。做好记录，整理逻辑，提前演练。

3. 注意事项

    不要跑题，讨论太宽泛太细都不可以，需要自己掌握分寸，也及时提醒别人

    必须争取发言。可以在关键节点做小总结，发言太少争取 Reporter ，再不行要争取补充。

    保持礼貌。不要一下子说好多，有时会导致后面没说的。

    不要抢角色。时间宝贵，角色只有一个，角色不重要，没有角色也可以做一个关键输出者。

    思路比结果重要。面试官更看重过程，而非你们产出的方案，毕竟方案最后是一个人来宣读。


专业面

1. 面试流程

    自我介绍。可以在此处展示亮点吸引面试官提问。

    挖掘经历。面试官会问简历内容，需要对简历熟悉，且能够表达自己经历中的思路——STAR。

    专业问题。与岗位相关问题，需要对这一行业这一公司这一岗位做足功课。

    反问。可以问自己有哪些不足，工作内容是什么可以做哪些准备。不要问薪资、环境等。

2. 注意事项

    如果是现场面，准备：服装、仪表、简历、材料、纸笔。

    回答问题少给自己挖坑，不懂的尽量不要说。

    不需要太紧张，面试本质是一场交流，但也不要太随意，体现诚意。

    不要抢话。回答问题停顿几秒，不懂也可以先说出思考过程。

    面试之后多做总结复盘。


HR 面

1. 主要问题

    性格相关。所以要对自己的性格有所了解，可以去做测试，也可以问问身边同学。

    求职偏好。包括意向薪资，看你最看重什么，如果问收到几个 offer 也尽量实话实说。

    稳定性。会问家在哪里，以后怎么规划，以及有没有对象等等。

2. 注意事项

    认真对待，不能轻视这一次面试，这一次更多的是在条件相似的候选人中筛选。

    可以询问薪资构成及五险一金情况，但最好不要做第一个说出具体薪资的人。


面试可以参考帅鸽学长的帖子。虽然是产品/运营，但也有共通之处。

多参加面试，先投递不大想去的公司实战练习，也可以参加一些模拟群面和单面。


面试之后

1. 感谢信。面试之后可以向 HR 和面试官表示感谢；如果没有过，也可以发邮件表达谢意。

2. 及时跟进。一般 HR 是不会反馈失败的面试结果的，但最好还是在一周后询问一下。

3. 总结复盘。
